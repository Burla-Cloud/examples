How it ran

Burla ships a Python function plus its imports to a fleet of containers running the standard python:3.12 image. The orchestrator sees them as a single iterator. The shared filesystem at ./shared is GCSFuse-mounted to a GCS bucket and visible from every worker, which is what makes this kind of slice-and-merge pipeline cheap.

Full writeup is on GitHub; Burla's docs are at docs.burla.dev.

What held up under bootstrap, and what did not

Median demand proxy per bucket with bootstrap 95% confidence intervals (1000 resamples, n ≥ 100 per bucket). A finding is rejected if any pair of bucket CIs overlaps. Two of four hypotheses survived.

Built on Burla. Caveats: demand proxy is reviews_per_month, not bookings. Photo coverage is whatever the public listing pages returned. Public Inside Airbnb data, no API access.
