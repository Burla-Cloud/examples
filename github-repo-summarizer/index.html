How it works

  1. Pull. BigQuery bigquery-public-data.github_repos — join files + contents + languages. One README per repo (biggest). Export as zstd parquet.
  2. Upload once. Stream the parquet to the cluster's shared filesystem (/workspace/shared).
  3. Fan out. remote_parallel_map(summarize_shard, 600 shards) on Burla — every worker reads its stripe of the parquet and runs the summarizer on ~2,000 READMEs each.
  4. Summarize. For each README: title, tldr, install method, category, badges, code blocks, token counts.
  5. Reduce. 16 buckets, each merges ~40 shard files in parallel. Top-K per category by "quality" score.
  6. Analyze locally. TF-IDF over 14 category "documents" surfaces distinctive keywords. Client-side search index sampled to 6,000 repos.

Source: Burla-Cloud on GitHub. Pipeline code in pipeline.py, scale.py, reduce.py, analysis.py.
